DocumentsProcessor

interface DocumentsProcessor

Responsible for performing processing on a Document from every Artifact in the pipeline. Is instantiated once, globally, and process is called for every Artifact, potentially from multiple threads. Implementations are responsible for thread-safety in the process method.

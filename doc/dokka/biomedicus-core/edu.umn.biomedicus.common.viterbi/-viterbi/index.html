firstOrder

 static fun <S : Any, Y : Any> firstOrder(emissionProbabilityModel: EmissionProbabilityModel<S, Y>, transitionProbabilityModel: TransitionProbabilityModel<S, S>, initialState: S): ViterbiProcessor<S, Y>

Creates a new first-order ViterbiProcessor. A first order viterbi processor only relies on the most recent state to compute transition probabilities, and only stores the most probable sequence for each possible most recent state for each step of computation.

secondOrder

 static fun <S : Any, Y : Any> secondOrder(emissionModel: EmissionProbabilityModel<S, Y>, transitionModel: TransitionProbabilityModel<S, Bigram<S>>, initialStates: Bigram<S>, bigramProvider: BiFunction<S, S, Bigram<S>>): ViterbiProcessor<S, Y>

Creates a new second-order ViterbiProcessor. A second order viterbi processor relies on the two most recent hidden states, and stores the most probable sequence for each possible permutation of the two most recent states at each step of computation.

BiomedicusScopes

class BiomedicusScopes

Contains the Guice scopes and contexts that store their scoped objects. There is one default scope:

  • Processor Scope: contains objects that are one instance per configured document processor. For example, any processor settings.
The primary method for running code in scopes are #createProcessorContext(Map).

Author
Ben Knoll

Since
1.6.0

Types

Context

 class Context

A context object, stores a ref to the context ThreadLocal and the map of objects to provide within the scope. #call(Callable) causes the current thread to enter the scope so that objects from the map are provided when scoped objects are needed.

providedViaSeeding

 static fun <T : Any> providedViaSeeding(): Provider<T>

A provider which represents an object that will be provided by seeding into the context via the map arguments on #createProcessorContext(Map). Modules can bind objects to this provider in order to satisfy injector dependencies prior to the creation of the scopes. The binding can only be injected inside a scope, and any provision request outside of a ContextScope will cause an IllegalStateException.
